# Google Antigravity AI 常用方法与技巧指南

Antigravity 是 Google DeepMind 开发的新一代 Agentic AI 编程助手。它不仅能回答问题，还能直接操作终端、编辑文件、管理项目，并像人类工程师一样与你“结对编程”。

以下是日常使用 Antigravity 的常用方法和进阶技巧。

## 1. 核心功能与日常使用

### 1.1 代码生成与编辑
- **从零构建**：可以直接告诉它“创建一个 Python 爬虫脚本”或“用 React 写一个登录页面”，它会自动创建文件并写入代码。
- **修改现有代码**：打开文件后，可以直接说“把这个函数的错误处理逻辑优化一下”或“给这段代码加详细注释”。
- **多文件重构**：Antigravity 可以理解项目上下文，支持跨文件的重构（例如“将 utils.py 中的函数移动到新的 helpers 模块，并更新所有引用”）。

### 1.2 终端操作
- **运行命令**：它可以直接在你的终端运行命令。你可以说“运行测试”、“安装 numpy 库”或“查看当前目录及其子目录结构”。
- **Git 管理**：它可以帮你执行 Git 操作，如 `git init`、`git commit`、`git push` 等，甚至自动生成 commit message。

### 1.3 调试与修复
- **报错排查**：遇到报错时，直接把错误信息贴给它，或者让它自己运行命令看报错。它会分析错误堆栈并提供修复方案。
- **Lint 修复**：可以结合 IDE 的 Lint 信息，让它一次性修复所有的格式或语法警告。

## 2. 进阶技巧 (Power User Tips)

### 2.1 Agentic Mode (任务模式)
当你要完成一个复杂的任务（如“重构整个鉴权模块”或“从头搭建一个全栈应用”）时，Antigravity 会进入 **Agentic Mode**。
- **Task List (任务清单)**：它会创建一个 `task.md`，列出所有需要做的步骤。
- **分步执行**：它会一步步执行，每完成一步都会勾选。
- **Artifacts (交付物)**：它会生成设计文档 (`implementation_plan.md`) 或验收文档 (`walkthrough.md`) 供你审查。

### 2.2 提供高质量上下文
- **打开相关文件**：Antigravity 会自动读取你当前打开（Active）和可见的文件。为了获得最佳结果，请打开与任务最相关的文件。
- **明确具体的指令**：“优化代码”不如“优化 calculate_total 函数的性能，减少循环嵌套”效果好。

### 2.3 交互与反馈
- **Human-in-the-loop**：在执行关键操作（如删除文件、Push 代码）前，它通常会寻求你的确认。
- **迭代修正**：如果它生成的代码不符合预期，不要直接放弃。告诉它哪里不对（例如“样式稍微有点乱，能不能用 Flexbox 居中？”），它会迅速修正。

## 3. 常用指令示例

| 场景 | 指令示例 |
| :--- | :--- |
| **新功能开发** | “在 `src/components` 下创建一个新的 `Button` 组件，支持 `primary` 和 `secondary` 变体。” |
| **代码解释** | “解释一下 `test.py` 第 10-20 行代码是在做什么？” |
| **Git 操作** | “提交当前更改，Commit message 设为 'Fix login bug'，然后推送到 main 分支。” |
| **测试** | “为 `utils.py` 中的 `parse_data` 函数写几个单元测试。” |

## 4. 总结

Antigravity 的强大之处在于它的 **Agentic（代理）** 能力。它不仅仅是一个聊天机器人，更是一个能使用工具、能感知环境、能执行任务的智能实体。把它当作你的结对编程伙伴，充分利用它的手动操作能力，将极大地提升你的开发效率。
